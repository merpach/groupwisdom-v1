GroupWisdom API

Send your team's data. Get AI-powered insights back.

POST items to a project → GroupWisdom runs analysis with Claude (Haiku for speed, Sonnet for depth) → insights are POSTed to your webhook_url and available via GET anytime.

JavaScript / TypeScript SDK

The easiest way to integrate GroupWisdom. Install once, call functions — no HTTP boilerplate required.

npm install @groupwisdom/sdk

Quick start

import GroupWisdom from '@groupwisdom/sdk'

const gw = new GroupWisdom({ apiKey: process.env.GROUPWISDOM_API_KEY })

// Create a project
const project = await gw.createProject('my-project')

// Send data — attributed to a specific person
await gw.ingest(project.id, {
  title: 'Q3 user research findings',
  content: 'Users in APAC want offline mode...',
  contributed_by: 'Sarah'
})

// Read insights when ready
const insights = await gw.listInsights(project.id)
insights.forEach(i => console.log(i.title, "—", i.body))

SDK reference

MethodDescription
new GroupWisdom({ apiKey })Create a client. Pass baseUrl to point at a self-hosted instance.
gw.createProject(name, { webhook_url? })Create a new project. Returns a Project object.
gw.listProjects()List all projects you have access to.
gw.getProject(projectId)Get a single project by ID.
gw.updateProject(projectId, { webhook_url })Update a project's webhook URL.
gw.ingest(projectId, item | item[])Send one or more items. Triggers analysis. Each item can include contributed_by.

Full TypeScript types are included — your editor will autocomplete every field.

Authentication

There are two types of API key:

Key typeFormatAccess
Personal keygw_...All your projects. Found in Developer API → Dashboard. Use for setup scripts and trusted backends.
Project keygw_proj_...One specific project only. Create via API or dashboard. Use in production apps — if it leaks, revoke it without affecting your account.
Authorization: Bearer gw_proj_your_project_key

Base URL

https://groupwisdom.up.railway.app/v1

All v1 endpoints are under /v1. The existing web app lives at /api and is unchanged.

Errors

All errors return JSON with an error field and a standard HTTP status code.

StatusMeaning
401Missing or invalid API key
400Bad request — missing required field
404Project not found or not yours
202Accepted — ingest queued, not yet processed

Ingest items

POST/v1/projects/:id/ingest

Send one item or an array of items. GroupWisdom queues analysis immediately — insights are delivered to your webhook_url within seconds and available via GET anytime.

Returns 202 Accepted — analysis is async.

Item fields

FieldTypeDescription
titlestringShort title or headline required*
contentstringFull text, notes, or description
urlstringLink to source
typestringnote · link · file · thought (default: note)

* At least one of title, content, or url is required per item.

Single item

curl -X POST https://groupwisdom.up.railway.app/v1/projects/proj_abc123/ingest \
  -H "Authorization: Bearer gw_yourkey" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Users keep abandoning checkout on mobile",
    "content": "Support tickets up 30% this month. Affects iOS more than Android.",
    "type": "note"
  }'

How the engine works

GroupWisdom runs a two-pass analysis every time new data comes in.

PassWhat happens
Pass 1 — Candidate scanA fast Haiku model reads all items in the project and identifies candidate insights — patterns, blind spots, connections, and questions that might be worth surfacing.
Pass 2 — Metacognitive evaluationA second model call reviews every candidate and decides: is this strong enough to surface? It assigns a confidence level, writes a concrete next action (do_next), notes any caveats, and flags whose perspective is missing (missing_voice). Weak candidates are suppressed entirely.

Only insights that survive both passes are saved. The result is fewer, higher-quality insights rather than a list of everything the model noticed.

Insights are generated automatically after every /ingest call. You can also trigger a fresh full analysis on demand — see Trigger analysis.

List insights

GET/v1/projects/:id/insights

Returns insights the engine has surfaced, newest first. Supports ?limit, ?offset, and ?kind filters.

By default returns a minimal shape — id, title, and body. Pass ?format=full to include metacognitive fields from Pass 2: kind, status, created_at, confidence, do_next, caveat, and missing_voice.

# Default (minimal)
curl "https://groupwisdom.up.railway.app/v1/projects/proj_abc123/insights" \
  -H "Authorization: Bearer gw_proj_yourkey"

# Full — includes metacognitive fields
curl "https://groupwisdom.up.railway.app/v1/projects/proj_abc123/insights?format=full" \
  -H "Authorization: Bearer gw_proj_yourkey"
// Default response
{
  "data": [
    {
      "id": "ins_xyz",
      "title": "Mobile payment friction is a recurring theme",
      "body": "Four of six items point to the same checkout drop-off."
    }
  ],
  "total": 3,
  "limit": 50,
  "offset": 0,
  "has_more": false
}

// With ?format=full
{
  "data": [
    {
      "id": "ins_xyz",
      "kind": "pattern",
      "title": "Mobile payment friction is a recurring theme",
      "body": "Four of six items point to the same checkout drop-off.",
      "status": "new",
      "created_at": "2026-07-15T14:30:00Z",
      "confidence": "high",
      "do_next": "Run a short usability session on the checkout flow with 3-5 users.",
      "caveat": "All data is from support tickets — may overrepresent frustrated users.",
      "missing_voice": "A product manager or designer with context on past payment work."
    }
  ],
  ...
}

Metacognitive fields

FieldValuesDescription
confidencehigh · medium · lowHow strongly the evidence supports this insight. High = 3+ independent data points; medium = 2; low = 1 or inferred.
do_nextstring or nullA concrete action the group should take. Null if the insight is purely observational.
caveatstring or nullA condition or limitation that could invalidate the insight if ignored.
missing_voicestring or nullA perspective or role not represented in the current data that would strengthen or challenge the insight.

These fields are always present in format=full responses — they may be null when not applicable.

You decide what to show your users. A simple card might render only title and body. A richer view might add do_next as a call-to-action button. A team dashboard might show all fields including confidence and missing voices.

Trigger analysis

POST/v1/projects/:id/analyze

Triggers a full two-pass analysis immediately, bypassing the automatic debounce timer. Useful when you want fresh insights on demand — for example, after a batch ingest or before showing results to a user.

Returns 202 Accepted immediately. Analysis runs in the background; poll List insights or wait for your webhook to receive results.

curl -X POST https://groupwisdom.up.railway.app/v1/projects/proj_abc123/analyze \
  -H "Authorization: Bearer gw_proj_yourkey"
{ "message": "Analysis started." }

Insight kinds

The engine generates insights across six categories:

KindDescription
connectionTwo items link in an unexpected or non-obvious way
patternMultiple data points imply a conclusion no one stated
blind_spotSomething obvious the group has not looked at
conflictTwo pieces of information contradict each other
questionSomething the group should be asking but is not
decisionSomething the group has decided, and what led to it

Project API keys

Project keys (gw_proj_...) are scoped to a single project. Use them in production apps instead of your personal key — if a project key leaks, you revoke just that key without touching your account or other projects.

Project keys can ingest data, read items, and read insights. They cannot create new projects or manage other keys — that requires your personal key.

Create a project key

POST/v1/projects/:id/keys

The full key is only returned once on creation. Store it immediately — subsequent GET /keys calls only show a redacted preview.

FieldTypeDescription
namestringLabel for this key (e.g. "Production", "Staging") required
curl -X POST https://groupwisdom.up.railway.app/v1/projects/proj_abc123/keys \
  -H "Authorization: Bearer gw_your_personal_key" \
  -H "Content-Type: application/json" \
  -d '{"name": "Production"}'
{
  "id": "key_xyz",
  "name": "Production",
  "key": "gw_proj_a1b2c3d4e5f6...",
  "created_at": "2026-06-30T10:00:00Z"
}

Revoke a key

DELETE/v1/projects/:id/keys/:keyId

Immediately invalidates the key. Any requests using it will receive a 401.

curl -X DELETE https://groupwisdom.up.railway.app/v1/projects/proj_abc123/keys/key_xyz \
  -H "Authorization: Bearer gw_your_personal_key"

Webhook setup

Set a webhook_url on any project (at creation or via PATCH). GroupWisdom will POST to that URL each time new insights are generated — typically within 3–5 seconds of an ingest call.

Your endpoint must return a 2xx status. Failed deliveries are not retried — poll /v1/projects/:id/insights as a fallback.

Use webhook.site to inspect payloads during development.

Webhook payload

Every webhook request includes an X-GroupWisdom-Signature header — a HMAC-SHA256 signature of the raw body using your webhook_secret. Always verify this before trusting the payload.

// Node.js — verify webhook signature
import { createHmac } from "crypto"

function verifyWebhook(rawBody, signature, secret) {
  const expected = "sha256=" + createHmac("sha256", secret).update(rawBody).digest("hex")
  return expected === signature
}

app.post("/groupwisdom-hook", express.raw({ type: "application/json" }), (req, res) => {
  const sig = req.headers["x-groupwisdom-signature"]
  if (!verifyWebhook(req.body, sig, process.env.GW_WEBHOOK_SECRET)) {
    return res.status(401).send("Invalid signature")
  }
  const payload = JSON.parse(req.body)
  // handle payload.insights ...
  res.sendStatus(200)
})

Your webhook_secret is returned when you set a webhook_url via PATCH. Store it securely — it is only shown once.

Payload shape

POST https://your-server.com/groupwisdom-hook
Content-Type: application/json
X-GroupWisdom-Signature: sha256=abc123...

{
  "event": "insights.created",
  "group_id": "proj_abc123",
  "insights": [
    {
      "id": "ins_xyz",
      "title": "Mobile payment friction is a recurring theme",
      "body": "Four of six items point to the same checkout drop-off."
    }
  ]
}

JavaScript / TypeScript example

Using the SDK (recommended):

import GroupWisdom from '@groupwisdom/sdk'

const gw = new GroupWisdom({ apiKey: process.env.GROUPWISDOM_API_KEY })

// Send items from multiple contributors
await gw.ingest("proj_abc123", [
  { title: "User retention dropped 12% in APAC", type: "note", contributed_by: "Sarah" },
  { title: "APAC pricing change rolled out last week", type: "note", contributed_by: "James" },
])

// Poll for insights — or set a webhook_url to receive them automatically
const insights = await gw.listInsights("proj_abc123")
insights.forEach(i => console.log(i.title, "—", i.body))

Using raw fetch (if you prefer no dependency):

const GW_KEY = process.env.GROUPWISDOM_API_KEY;
const BASE   = "https://groupwisdom.up.railway.app/v1";

await fetch(`${BASE}/projects/proj_abc123/ingest`, {
  method: "POST",
  headers: { "Authorization": `Bearer ${GW_KEY}`, "Content-Type": "application/json" },
  body: JSON.stringify({ items: [
    { title: "User retention dropped 12% in APAC", type: "note", contributed_by: "Sarah" },
  ]}),
});

Python example

import os, requests

GW_KEY = os.environ["GROUPWISDOM_API_KEY"]
BASE   = "https://groupwisdom.up.railway.app/v1"
HEADERS = {"Authorization": f"Bearer {GW_KEY}", "Content-Type": "application/json"}

def ingest(project_id, items):
    r = requests.post(f"{BASE}/projects/{project_id}/ingest",
                      json={"items": items}, headers=HEADERS)
    r.raise_for_status()
    return r.json()

def get_insights(project_id, kind=None):
    params = {"kind": kind} if kind else {}
    r = requests.get(f"{BASE}/projects/{project_id}/insights",
                     params=params, headers=HEADERS)
    r.raise_for_status()
    return r.json()

# Usage
ingest("proj_abc123", [
    {"title": "User retention dropped 12% in APAC", "type": "note"},
    {"title": "APAC pricing change rolled out last week", "type": "note"},
])

insights = get_insights("proj_abc123", kind="pattern")
for ins in insights:
    print(f"[{ins['kind']}] {ins['title']}: {ins['body']}")
