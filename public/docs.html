GroupWisdom API

Send your team's data. Get AI-powered insights back.

POST items to a project → GroupWisdom runs analysis with Claude (Haiku for speed, Sonnet for depth) → insights are POSTed to your webhook_url and available via GET anytime.

Authentication

All requests require your personal API key passed as a Bearer token:

Authorization: Bearer gw_your_api_key

Find your key in the app under Developer API → Dashboard, or at GET /api/me.

Base URL

https://your-groupwisdom-url.com/v1

All v1 endpoints are under /v1. The existing web app lives at /api and is unchanged.

Errors

All errors return JSON with an error field and a standard HTTP status code.

StatusMeaning
401Missing or invalid API key
400Bad request — missing required field
404Project not found or not yours
202Accepted — ingest queued, not yet processed

Create project

POST/v1/projects
FieldTypeDescription
namestringProject name required
webhook_urlstringURL to receive insight callbacks optional
curl -X POST https://your-url.com/v1/projects \
  -H "Authorization: Bearer gw_yourkey" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Q3 User Research",
    "webhook_url": "https://your-server.com/groupwisdom-hook"
  }'
{
  "id": "proj_abc123",
  "name": "Q3 User Research",
  "created_at": "2026-06-26T14:22:00Z",
  "webhook_url": "https://your-server.com/groupwisdom-hook",
  "counts": { "items": 0, "insights": 0 }
}

List projects

GET/v1/projects

Returns all projects you have access to.

curl https://your-url.com/v1/projects \
  -H "Authorization: Bearer gw_yourkey"

Get project

GET/v1/projects/:id

Returns a single project with current item and insight counts.

Update project

PATCH/v1/projects/:id

Update the webhook URL or name. Only fields you send are changed.

FieldTypeDescription
webhook_urlstring | nullSet to null to remove the webhook
curl -X PATCH https://your-url.com/v1/projects/proj_abc123 \
  -H "Authorization: Bearer gw_yourkey" \
  -H "Content-Type: application/json" \
  -d '{"webhook_url": "https://new-server.com/hook"}'

Ingest items

POST/v1/projects/:id/ingest

Send one item or an array of items. GroupWisdom queues analysis immediately — insights are delivered to your webhook_url within seconds and available via GET anytime.

Returns 202 Accepted — analysis is async.

Item fields

FieldTypeDescription
titlestringShort title or headline required*
contentstringFull text, notes, or description
urlstringLink to source
typestringnote · link · file · thought (default: note)

* At least one of title, content, or url is required per item.

Single item

curl -X POST https://your-url.com/v1/projects/proj_abc123/ingest \
  -H "Authorization: Bearer gw_yourkey" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Users keep abandoning checkout on mobile",
    "content": "Support tickets up 30% this month. Affects iOS more than Android.",
    "type": "note"
  }'

Bulk ingest

curl -X POST https://your-url.com/v1/projects/proj_abc123/ingest \
  -H "Authorization: Bearer gw_yourkey" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      {"title": "Mobile checkout drop-off up 30%", "type": "note"},
      {"title": "Competitor added Apple Pay last week", "type": "note"},
      {"url": "https://example.com/analytics-report", "type": "link"},
      {"title": "Team decision: prioritise mobile payment flow", "type": "thought"}
    ]
  }'
{
  "accepted": 4,
  "items": [
    {"id": "item_001", "title": "Mobile checkout drop-off up 30%", "type": "note"},
    ...
  ],
  "message": "Items queued for analysis. Insights will be POSTed to your webhook_url when ready."
}

List items

GET/v1/projects/:id/items

Returns all items in a project, newest first.

curl https://your-url.com/v1/projects/proj_abc123/items \
  -H "Authorization: Bearer gw_yourkey"

List insights

GET/v1/projects/:id/insights

Returns all insights the engine has surfaced, newest first. Filter by kind with ?kind=pattern.

curl "https://your-url.com/v1/projects/proj_abc123/insights?kind=pattern" \
  -H "Authorization: Bearer gw_yourkey"
[
  {
    "id": "ins_xyz",
    "kind": "pattern",
    "title": "Mobile payment friction is a recurring theme",
    "body": "Four of six items point to the same checkout drop-off. A shared pattern is emerging that no one has named yet.",
    "status": "acknowledged",
    "created_at": "2026-06-26T14:30:00Z"
  }
]

Insight kinds

The engine generates insights across six categories:

KindDescription
connectionTwo items link in an unexpected or non-obvious way
patternMultiple data points imply a conclusion no one stated
blind_spotSomething obvious the group has not looked at
conflictTwo pieces of information contradict each other
questionSomething the group should be asking but is not
decisionSomething the group has decided, and what led to it

Webhook setup

Set a webhook_url on any project (at creation or via PATCH). GroupWisdom will POST to that URL each time new insights are generated — typically within 3–5 seconds of an ingest call.

Your endpoint must return a 2xx status. Failed deliveries are not retried — poll /v1/projects/:id/insights as a fallback.

Use webhook.site to inspect payloads during development.

Webhook payload

POST https://your-server.com/groupwisdom-hook
Content-Type: application/json

{
  "event": "insights.created",
  "group_id": "proj_abc123",
  "insights": [
    {
      "id": "ins_xyz",
      "kind": "pattern",
      "title": "Mobile payment friction is a recurring theme",
      "body": "Four of six items point to the same checkout drop-off."
    }
  ]
}

JavaScript example

const GW_KEY = process.env.GROUPWISDOM_API_KEY;
const BASE   = "https://your-groupwisdom-url.com/v1";

async function ingest(projectId, items) {
  const res = await fetch(`${BASE}/projects/${projectId}/ingest`, {
    method: "POST",
    headers: {
      "Authorization": `Bearer ${GW_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ items }),
  });
  return res.json();
}

async function getInsights(projectId) {
  const res = await fetch(`${BASE}/projects/${projectId}/insights`, {
    headers: { "Authorization": `Bearer ${GW_KEY}` },
  });
  return res.json();
}

// Usage
await ingest("proj_abc123", [
  { title: "User retention dropped 12% in APAC", type: "note" },
  { title: "APAC pricing change rolled out last week", type: "note" },
]);

// Or set a webhook and receive insights automatically

Python example

import os, requests

GW_KEY = os.environ["GROUPWISDOM_API_KEY"]
BASE   = "https://your-groupwisdom-url.com/v1"
HEADERS = {"Authorization": f"Bearer {GW_KEY}", "Content-Type": "application/json"}

def ingest(project_id, items):
    r = requests.post(f"{BASE}/projects/{project_id}/ingest",
                      json={"items": items}, headers=HEADERS)
    r.raise_for_status()
    return r.json()

def get_insights(project_id, kind=None):
    params = {"kind": kind} if kind else {}
    r = requests.get(f"{BASE}/projects/{project_id}/insights",
                     params=params, headers=HEADERS)
    r.raise_for_status()
    return r.json()

# Usage
ingest("proj_abc123", [
    {"title": "User retention dropped 12% in APAC", "type": "note"},
    {"title": "APAC pricing change rolled out last week", "type": "note"},
])

insights = get_insights("proj_abc123", kind="pattern")
for ins in insights:
    print(f"[{ins['kind']}] {ins['title']}: {ins['body']}")
